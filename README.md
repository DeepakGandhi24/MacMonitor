MacMonitor
A native macOS SwiftUI application for monitoring Mac fleet performance using JAMF Pro. MacMonitor collects real-time CPU, memory, disk, battery and network metrics from managed Macs and displays them in a unified dashboard.

Screenshots
Dashboard · Devices · Overview · Export

Features
JAMF Pro Integration — fetches full device inventory via OAuth2 API
Real-time Metrics — CPU usage, memory pressure, disk space, battery health, network rates
Fleet Overview — combined stats with gauges, distribution charts and alerts
Per-device Detail — top processes, load averages, memory breakdown
Agent-based Collection — lightweight Python agent runs as a launchd daemon on each Mac
CSV Export — export all devices or reporting devices for auditing
Microsoft Teams — post fleet health reports via Incoming Webhook
ServiceNow — auto-create incidents for devices with performance issues
Keychain Storage — credentials stored securely, never in plain text

Requirements
macOS 13.0 (Ventura) or later
Xcode 15 or later
JAMF Pro with OAuth2 API client configured
Python 3 on managed Macs (pre-installed on macOS)

Installation
Build from source
git clone https://github.com/DeepakGandhi24/MacMonitor.git
cd MacMonitor
open MacMonitor.xcodeproj
In Xcode select your team under Signing & Capabilities
Press ⌘R to build and run

Setup
1 — Connect to JAMF Pro
Launch MacMonitor
Click the ⚙️ gear icon → JAMF tab
Enter your JAMF Pro URL, Client ID and Client Secret
Tick Save credentials to Keychain if you want them remembered
Click Test Auth → wait for ✅
Click Fetch Devices → all your Macs appear in the sidebar
2 — Create a JAMF Pro API Client
Log into JAMF Pro → Settings → API Roles and Clients
Create an API Role with these permissions:
Read Computers
Read Computer Inventory Collection
Create an API Client → assign the role → generate a Client Secret
3 — Install the Agent on the Dashboard Mac
Go to Settings → Agent tab
Enter this Mac's IP address (the Mac running MacMonitor)
Click Install & Start Agent → enter your admin password
The agent starts automatically and posts metrics every 5 minutes
4 — Deploy to All Managed Macs via JAMF
Go to Settings → Agent tab
Click Export for JAMF Deployment
Files are saved to ~/Desktop/MacMonitorAgent/
In JAMF Pro → Scripts → New → paste the contents of jamf_deploy.sh
Create a Policy → add the script → scope to All Computers
Set trigger to Recurring Check-in
Once deployed, each Mac's agent phones home to the dashboard automatically.

How the Agent Works
Mac boots
  → launchd reads com.macmonitor.agent.plist
  → starts agent.py automatically as a background daemon
  → agent collects metrics every 5 minutes:
      CPU usage · Memory · Disk · Battery · Network · Top processes
  → POSTs JSON to dashboard Mac on port 9876
  → if agent crashes → launchd restarts it automatically
The agent script is generated by the app with your dashboard Mac's IP pre-configured. Generate it via Settings → Agent → Export for JAMF Deployment.

Agent Deployment
Important: Always generate the agent via the app — Settings → Agent → Export for JAMF Deployment. This ensures your dashboard Mac's IP is correctly embedded in the script.
The Agent/agent.template.py in this repo is a reference only. It does not contain a pre-configured IP.

Integrations
Microsoft Teams
In Teams → Channel → ··· → Connectors → Incoming Webhook
Copy the webhook URL
In MacMonitor → Settings → Integrations → Teams
Paste the URL → click Send Fleet Report Now
ServiceNow
In MacMonitor → Settings → Integrations → ServiceNow
Enter your instance URL, username and password
Devices with performance issues appear automatically
Click Create Incident per device or Create All Incidents

Privacy & Security
No data leaves your network except to your own JAMF Pro server
All credentials stored in macOS Keychain (kSecAttrAccessibleWhenUnlockedThisDeviceOnly)
Agent communicates only with the dashboard Mac's IP on port 9876
No third-party services, no telemetry, no cloud storage

Troubleshooting
Problem
Solution
No devices loading
Check JAMF credentials in Settings → JAMF → Test Auth
Dashboard shows only 1 device
Credentials not saved — enter them in Settings → JAMF → Fetch Devices
Agent shows 0% CPU
Agent needs updating — reinstall via Settings → Agent → Install & Start
Network shows 0 KB/s
No active traffic during measurement — normal when idle
Agent not posting
Check tail -f /Library/MacMonitor/agent.log
